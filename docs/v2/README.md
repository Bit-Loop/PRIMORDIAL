# V2 Planning Artifacts

This directory holds future-version planning artifacts that are not loaded by Primordial V1 runtime code.

- `reverse_engineering_framework.md`: control-plane framework for binary exploitation research, software reverse engineering, embedded firmware analysis, hardware reverse engineering, and FPGA reverse engineering.
- `reverse_engineering_tooling.yaml`: structured V2-only tooling taxonomy with controls, domains, toolchains, risks, and expected evidence types.
- `reverse_engineering_test_plan.md`: fixture, policy, runner, hardware-bench, and release acceptance criteria.

Any future V2 integration should explicitly move selected material into runtime catalogs, storage models, policy gates, and tests through a dedicated implementation milestone.
